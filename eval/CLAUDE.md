# Evaluation Module Map

## Members

- `visual-contract.md`: durable rendering contract for the Siri-style effect.
- `work-contract.md`: done definition for this implementation pass.
- `evidence-report.md`: final report for screenshots, checks, and residual review needs.
- `evidence/`: screenshots, logs, and generated QA artifacts.

## Boundaries

- Contracts define quality expectations.
- Evidence records what was actually checked.
- Baselines must not be refreshed to hide regressions.
